XRPL Glossary
← Back to Main Page

Issuer Address

The wallet that *creates* the token. XRPL does not use contract addresses. Every asset originates from its issuer.

Currency Code

The token identifier (ticker). Can be 3-character or a full 160-bit hex code.

Trustline

A permission line allowing a wallet to hold a token from a specific issuer. No trustline means the token cannot enter the wallet.

AMM (Automated Market Maker)

XRPL’s built-in liquidity engine. Enables swaps, trading, and pathfinding without smart contracts.

Liquidity Pool

A pool of XRP + a token. Defines price, depth, volatility resistance, and swap execution.

LP Tokens

Represent your percentage share in a liquidity pool. Earns a portion of fees and arbitrage rewards.

Routed Trade

A swap executed across multiple pools to get the best price. Charts often miss routed volume.

Pathfinding

The algorithm XRPL uses to locate optimal trade routes through XRP, stablecoins, or other asset pairs.

DEX (On-Ledger)

The native XRPL decentralized exchange. No smart contracts, no dApps — everything is on-ledger.

Rippling

Allows assets with the same currency code to flow through trustlines. Usually disabled unless intentionally needed.

NFToken (XLS-20)

XRPL’s native NFT standard. Fast, cheap, on-ledger, and clean.

Memos

Metadata embedded in transactions — for instructions, IDs, or verification proofs.

Escrow

Time or condition-locked XRP. Used for delayed releases, guarantees, or timed payouts.

Clawback

An optional compliance flag allowing issuers to reclaim tokens under specific regulated conditions.

AccountSet

A ledger-level configuration transaction used to modify account settings.

Hooks (Testnet)

Low-level programmable logic for conditional behavior. Not live on XRPL mainnet yet.
