XRPL Glossary
← Back to Main Page

Issuer Address

The account that creates and issues the token. XRPL does not use contract addresses. Every asset is tied directly to its issuer.

Currency Code

The token ticker. Can be short (e.g., RAD, BAG) or a full 160-bit hexadecimal code.

Trustline

A permission line allowing a wallet to hold a specific token from a specific issuer. Without it, the token cannot be received.

AMM (Automated Market Maker)

The ledger’s built-in liquidity and swap engine. No smart contracts required.

Liquidity Pool

A pool containing XRP + another asset. Determines price stability, depth, and execution quality.

LP Tokens

Tokens representing your ownership percentage of a liquidity pool, earning a share of its fees.

Routed Trade

A swap spanning multiple liquidity paths to achieve the best price. Often misunderstood by new users.

Pathfinding

The XRPL routing algorithm that finds optimal swap routes through XRP, stablecoins, or other assets.

DEX (On-Ledger)

The native decentralized exchange built directly into XRPL. Everything happens on-ledger.

Rippling

Allows value with the same currency code to traverse trustlines. Usually disabled unless used intentionally.

NFToken (XLS-20)

The XRPL NFT standard. Ultra-fast, ultra-cheap, on-ledger.

Memos

Metadata attached to transactions for verification, instructions, or proofs.

Escrow

Condition or time-locked XRP. Perfect for delayed releases or contracts without smart contracts.

Clawback

An optional issuer-level compliance flag allowing token retrieval under regulated circumstances.

AccountSet

Transaction type used to modify account configuration and flags.

Hooks (Testnet)

Advanced programmable logic proposed for XRPL — not enabled on mainnet yet.
