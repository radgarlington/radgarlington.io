XRPL Glossary
← Back to Ledger

Issuer Address

The account that creates and issues the token. XRPL does not use contract addresses.

Currency Code

The token ticker. Either short (e.g., RAD, BAG) or 160-bit hex.

Trustline

A required permission line that allows a wallet to hold a token from the issuer.

AMM (Automated Market Maker)

XRPL’s native liquidity engine. Enables swaps without smart contracts.

Liquidity Pool

A pool containing XRP + an issued token. Determines price and stability.

LP Tokens

Your share of a liquidity pool, earning proportional fees.

Routed Trade

A swap that uses multiple liquidity paths for best execution.

Pathfinding

The XRPL algorithm that determines optimal trade routes.

DEX (On-Ledger)

The built-in decentralized exchange all XRPL assets trade on.

Rippling

Allows value to pass between trustlines with the same currency code.

NFToken (XLS-20)

XRPL’s on-ledger NFT standard.

Memos

Small metadata fields attached to transactions.

Escrow

Time or condition-locked XRP for delayed release or guarantees.

Clawback

Optional issuer compliance feature allowing token reclamation.

AccountSet

Transaction for modifying account flags and configuration.

Hooks (Testnet)

Low-level programmable XRPL logic not yet live on mainnet.
